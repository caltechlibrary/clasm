STATUS: Phase 5b is implemented, tested, and committed. Working tree is

What happened this session
---------------------------

1. Reviewed DESIGN.md / DECISIONS.md / PLAN.md against the actual code and
   found several gaps:
   - PLAN.md's checkboxes were stale (Phases 0-7 were already implemented
     in ec2_ami_manager.bash, not just Phase 0).
   - The whole BATS test suite was broken: all .bats files hardcoded a
     load path to the pre-rename `aws_tools` directory. Fixed to a
     relative `load 'lib/test_helper'`.
   - check_ami.bash and check_ec2_instances.bash duplicated
     list_ec2_instances()/list_amis() with no unique functionality.
     Retired (deleted, committed).
   - ami_copy.bash (merged in from a separate repo before this project's
     design docs existed) duplicated the "create AMI from instance"
     feature but had real capabilities ec2_ami_manager.bash's Phase 5
     lacked: volume-size time estimates, prior-snapshot detection, an SSM
     fstrim pre-snapshot optimization, unbounded creation polling, and
     Invenio-RDM-specific crash-consistency guidance. Also found that
     Phase 5's post_ami_creation_actions() had a 600-second timeout that
     would fail on real Invenio RDM AMI sizes (20-60+ min per
     ami_copy_basic_steps.md's own timing table).

2. Decided (recorded in DECISIONS.md): port ami_copy.bash's capabilities
   into Phase 5's multi-region workflow (new PLAN.md Phase 5b), THEN
   retire ami_copy.bash — not before, so nothing is silently dropped.

3. Implemented Phase 5b, TDD style, in ec2_ami_manager.bash:
   - estimate_ami_creation_time(gb) — time-estimate buckets, ported from
     ami_copy.bash's estimate_time()
   - gather_volume_info(instance_id, region) — sums attached volume
     sizes, detects prior snapshots
   - aws_ssm() — new wrapper alongside aws_ec2()
   - check_ssm_availability(), run_fstrim_via_ssm(),
     offer_fstrim_before_snapshot() — SSM fstrim step
   - format_elapsed(seconds) + rewrote post_ami_creation_actions() to
     poll unboundedly (AMI_POLL_INTERVAL env var, default 30s) instead of
     giving up after 600s; explicit failed-state short-circuit
   - select_instance_for_ami()'s running-instance warning now carries the
     Postgres/OpenSearch/Redis/Docker crash-consistency guidance from
     ami_copy_basic_steps.md
   All of the above are wired into select_instance_for_ami(), which is
   called from create_ami_from_instance_workflow() (Phase 5's entry
   point).

4. tests/test_create_ami.bats — 27 new tests covering all of the above,
   including two that exercise the *interactive* prompts for real (not
   stubbed) using bash heredoc-stdin redirection on `run`:
     run select_instance_for_ami <<< "y"
   This pattern works in this BATS setup and is more honest than the
   stub-everything pattern used elsewhere in the suite — worth reusing
   for Phase 6/7 tests instead of skip-with-TODO.

5. Updated PLAN.md (Phase 5b work items checked off except retirement),
   TODO.md, tests/README.md (accurate coverage table), DESIGN.md (file
   structure section, noted ami_copy.bash as "to be retired").
