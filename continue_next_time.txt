Today's work, in order: Full-height Menu Tier (20.26); Launch
Templates (20.27/20.28) -- create/show/sync/promote/delete built

applyOneTagChange to take a pluggable apply closure for this). All
five Tag Management kinds are real-AWS confirmed.

Three real bugs found only via live usage, not tests, fixed same day:
(1) a context-cancellation bug that broke launching an instance from a
launch template (WaitUntilRunning reused an already-cancelled timeout
context); (2) AWS rejecting a "v"-prefixed launch template version
selector outright (normalizeVersionSelector fixes it -- this is what
made Promote look silently broken); (3) "Show tags" appearing to do
nothing on a real terminal -- root cause: the new full-height action
Select (Phase 20.26) scrolled the plain-printed tag listing out of
view the instant it rendered, fixed by embedding the tags directly in
the Select's own Description (actionMenuDescription). This last one is
a general lesson worth remembering: any call site that prints plain
text immediately before a full-height Menu-tier Select is at risk of
the same silent-scroll bug -- check for that pattern first if a future
"the screen looks unchanged" report comes in.

TUI_REFERENCE.md (new this session, mid-week) has verified ANSI box
mockups per tier plus a screen-map tree -- read it before touching
chrome, it was built specifically to stop resize-behavior confusion
between the Menu/Picker/List/Manager tiers (a multi-round investigation
this session concluded was stale-binary testing, not a real bug, once
resolved).
