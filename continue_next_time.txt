What shipped this session
--------------------------------------------------------------------

- `internal/filemanager`, a new package holding a scoped `bubbletea`
  `Model` -- single-pane (bucket only) or double-pane (bucket + a
  linked local directory), sharing one Model/Update/View. Covers:
  navigation (arrow keys, Enter/Backspace), tagging (`Space`/`*`),
  per-level substring filter (`f`/`/`), Upload/Download/Delete/Show
  metadata actions with a Confirm/ConfirmDestructive/progress modal
  overlay, mid-session link/unlink (`l`), a dedicated Sync action
  (`S`/`:sync` -- whole-tree diff against the bucket, added mid-session
  after the user confirmed it was worth building rather than relying on
  manual tag-and-act), Find (`F`/`:find <pattern>`, recursive
  glob-on-basename), and a `:`-prefixed command line mirroring every
  hotkey.
- `internal/s3diff`, a new package holding the key+size diff logic
  (`Compute`/`WalkLocalTree`/`ListAllBucketObjects`/`UploadFile`/
  `ValidateLocalDirectory`) extracted from the now-deleted
  `bucket_sync.go` -- both `internal/workflow` (Delete Bucket doesn't
  need it, but the pattern of "extract instead of duplicate" applied
  here too) and `internal/filemanager` depend on it, avoiding a
  workflow<->filemanager import cycle while keeping the logic in one
  place.
- `s3:GetObject` added to `S3API` (+ logging wrapper + fake) --
  completes Create/Update/Read/Delete parity for S3.
- `github.com/charmbracelet/huh` added as a dependency for the file
  manager's own bucket-selection pre-flight
  (`internal/workflow/object_browser.go`); every other S3 wizard is
  unaffected, still on termlib.
- Retired: `bucket_browse.go`'s single-object wizard,
  `bucket_delete_objects.go`'s prefix-delete wizard, and
  `bucket_sync.go`'s directory-sync wizard -- all three are now
  reachable from one "Browse & Manage Objects" menu entry
  (`s3_menu.go`, `main.go` updated). `bucket_browse.go` keeps only
  `listBucketObjectsWithPrefix` (Delete Bucket's empty-check still
  needs it).
- Tests use `github.com/charmbracelet/x/exp/teatest` -- resolves
  PLAN.md's open "how do you test a bubbletea Model" question. Caught
  one real data race via `go test -race` (a background action
  goroutine mutating pane state directly instead of only sending over
  its progress channel) -- fixed; see DECISIONS.md for the general rule
  this sets for any future action added to this Model.
